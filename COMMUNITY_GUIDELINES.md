# Community Guidelines

Welcome to the AOU-RTL community! These guidelines help ensure a positive and productive environment for all contributors.

## Code of Conduct

All contributors must follow our [Code of Conduct](CODE_OF_CONDUCT.md).

**In summary:**
- Be respectful and inclusive
- Welcome newcomers
- Assume good intentions
- Give and receive feedback gracefully
- No harassment, discrimination, or trolling

**Enforcement:** Violations can be reported to ospo@tenstorrent.com

---

## Communication Channels

### GitHub Issues

**Use for:**
- Bug reports
- Feature requests
- Technical discussions

**Best practices:**
- Search existing issues before creating new ones
- Use appropriate issue templates
- Provide clear, detailed descriptions
- Include simulator version, OS, and relevant log files
- Do not disclose confidential materials

### GitHub Pull Requests

**Use for:**
- RTL / testbench contributions
- Documentation improvements
- Technical reviews

**Best practices:**
- Link to the related issue
- Fill out the PR template completely
- Keep PRs focused and reasonably sized
- Respond to review feedback promptly

### Discord Community

**Join at:** https://discord.gg/7gzmYKRX

**Use for:**
- Real-time help and discussions
- Community building
- Informal questions
- Announcements
- Getting to know other contributors

**Best practices:**
- Search chat history before asking
- Use appropriate channels
- Be patient - community support is volunteer-driven
- Help others when you can
- Do not disclose confidential materials

## Getting Help

### For Development Questions

1. **Check existing documentation:**
   - [CONTRIBUTING.md](CONTRIBUTING.md) - How to contribute, branch naming, coding standards
   - [DOC/integration_guide/integration_guide.md](DOC/integration_guide/integration_guide.md) - Integration guide for `AOU_CORE_TOP`
   - [DOC/csr/aou-core-csrs.md](DOC/csr/aou-core-csrs.md) - Register map reference
   - [DOC/MAS/aou_core_mas.md](DOC/MAS/aou_core_mas.md) - Micro-Architecture Specification

2. **Search existing issues:**
   - Someone may have asked before
   - Check closed issues too

3. **Ask on Discord:**
   - Join at https://discord.gg/7gzmYKRX
   - Real-time responses from the community

4. **Open an issue:**
   - Use the "Question" label
   - Provide context and what you've already tried
   - Link to relevant code or documentation

### For Bug Reports or Feature Requests

Open a [GitHub Issue](https://github.com/tenstorrent/aou-rtl/issues) and choose an appropriate label:
- **Bug** - For RTL, testbench, or script issues
- **Feature Request** - For new functionality or enhancements
- **Question** - For general questions

### Response Times

- **Critical bugs:** 24-48 hours
- **Regular issues:** 3-7 days
- **Feature requests:** Triaged within 2 weeks
- **Discord questions:** Often answered within hours (community-driven)

**Note:** Response times are best-effort. This is an open-source project maintained by volunteers and Tenstorrent staff.

## Review Etiquette

### For Authors

- Respond to feedback constructively
- Ask questions if feedback is unclear
- Keep PRs reasonably sized (easier to review)
- Update the PR description if scope changes
- Be patient - reviews take time

### For Reviewers

- Be respectful and constructive
- Explain the "why" behind feedback
- Distinguish between blocking issues and suggestions
- Acknowledge good work
- Respond in a reasonable timeframe
